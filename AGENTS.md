# Repository Guidelines

このファイルは AI エージェント向けの **Single Source of Truth** です。
コード生成・レビュー・ドキュメント更新時に必ず従ってください。

## AI エージェントへの指示

- コードは授業教材として使用するため、**シンプルさ・可読性を最優先**すること。
- コメントや補足説明は **日本語** で記述すること。
- **外部ライブラリを導入しないこと**。Java 標準ライブラリのみ使用可。
- 既存コードは「1ファイル = 1つの独立した実行例」という構成なので、その形を崩さないこと。
- 学習用に **コンパイルエラーになる例をあえて有効化して置く** ことがある。意図のある教材コードは安易に修正・削除しないこと。

## プロジェクト構成とモジュール

- **言語**: Java（標準ライブラリのみ、外部依存なし）
- **IDE**: IntelliJ IDEA（`.iml` プロジェクト）
- **ビルドツール**: なし（`javac` / `java` で直接コンパイル・実行）
- **ソースルート**: `src/`（デフォルトパッケージ）

現在の `src/` には、主に次の教材コードが入っています。

```text
src/
├── Main.java           # Hello, World!
├── Test0201.java       # print / println の違い
├── Test0202.java
├── ...
├── Test0215.java       # 文字列・真偽値・文字リテラルなど
├── Arith01.java        # 整数の算術演算
├── Arith02.java        # 実数の算術演算
├── Arith03.java        # int と double の混在演算
├── Concat01.java       # 文字列連結の基本
├── Concat02.java       # 連結順序と式の評価
└── Concat03.java       # int / char / String の違い
```

## ビルド・テスト・開発コマンド

```bash
# 単一ファイルをコンパイル
javac src/<ClassName>.java

# 単一ファイルを実行
java -cp src <ClassName>
```

- テストフレームワークは未導入。
- すべての `.java` ファイルは個別に確認する教材プログラムとして扱う。
- 通常の実行例に加えて、コンパイルエラーを確認するためのファイルも含む。
- 一部のファイルは教材用に **意図的なコンパイルエラーを含む** ため、リポジトリ全体の一括コンパイルは前提にしない。

## コーディング規約と命名

- 各ファイルは `public static void main(String[] args)` を持つ独立したプログラムにする。
- パッケージ宣言は書かない（デフォルトパッケージ）。
- 授業用教材なので、複雑な抽象化や過度な共通化は避ける。
- 誤った記述例を示す場合は、日本語で理由を書き、教材の意図に応じて **有効なまま置く** か **コメントアウトして残す**。

現在使われているファイル命名規則:

| プレフィックス | 用途 | 例 |
|--------------|------|-----|
| `Main` | 最初の実行例 | `Main.java` |
| `TestNNNN` | 節番号付きの基本例題 | `Test0203.java` |
| `ArithNN` | 算術演算の例 | `Arith01.java` |
| `ConcatNN` | 文字列連結の例 | `Concat02.java` |

新しいソースファイルを追加する場合も、この流れに合わせて **教材の分類が分かる名前** にすること。
